# 2026-06-05

## 작업 개요

GitHub Actions 자동 업데이트 흐름에서 발생한 Gemini API 실패, Actions 런타임 경고, 공공데이터 수집 주기 문제를 정리하고 자동화 안정성을 개선했습니다.

## 주요 작업

- Gemini 블로그 생성 단계에서 `429 Too Many Requests`가 발생하는 원인을 조사했습니다.
- Gemini 응답 본문을 통해 `RESOURCE_EXHAUSTED`와 `prepayment credits are depleted` 상태를 확인했고, AI Studio 선불 크레딧 부족 문제로 정리했습니다.
- `scripts/generate-blog-post.js`에 Gemini `429`/`5xx` 재시도 로직을 추가했습니다.
- 블로그 생성 실패 시 `BLOG_GENERATION_FAILURE_MODE=keep-existing` 설정이면 기존 파일을 유지하고 배포 흐름을 계속 진행하도록 했습니다.
- GitHub Actions의 블로그 생성 step에 Gemini 재시도 및 실패 유지 설정을 추가했습니다.
- GitHub Actions의 Node.js 20 deprecation warning 원인을 확인하고, `actions/checkout`과 `actions/setup-node`를 Node 24 기반 major 버전으로 업데이트했습니다.
- 행사 데이터 수집의 `EVENTS_START_DATE` 고정값을 제거하고, 한국시간 오늘 날짜를 기본 시작일로 쓰도록 변경했습니다.
- 일반음식점 인허가 데이터 수집은 조회량이 커서 매일 실행하지 않고, 한국시간 매월 1일 또는 수동 실행에서만 실행되도록 워크플로 조건을 추가했습니다.

## 메모

- Gemini API의 Cloud Billing 예산과 AI Studio prepay credit은 별도 개념입니다. 이번 실패는 예산 한도 초과가 아니라 선불 크레딧 소진 상태였습니다.
- Gemini 선불 크레딧 충전 후에는 새 GitHub Actions 실행부터 블로그 생성이 다시 시도됩니다.
- 행사 시작일 동적화와 음식점 월 1회 수집 변경은 로컬 작업트리에 남아 있으며, 별도 커밋/푸시가 필요합니다.
