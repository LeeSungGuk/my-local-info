# 2026-04-14

## 작업 개요

지원금/혜택 영역을 공공데이터포털 기준으로 재구성하고, `benefits` 전용 데이터 파이프라인과 화면 구조를 분리했습니다.

## 주요 작업

- 공공데이터포털 `대한민국 공공서비스(혜택) 정보`를 기준으로 혜택 수집 스크립트를 추가했습니다.
- 사용 API는 `serviceList`, `serviceDetail`, `supportConditions` 3개로 고정했습니다.
- 혜택 데이터 저장 구조를 `public/data/benefits/index.json` + `public/data/benefits/items/*.json` 형태로 분리했습니다.
- 혜택 전용 로더를 추가하고, 홈 화면과 상세 페이지가 새 혜택 데이터 구조를 읽도록 변경했습니다.
- `/benefits` 목록 페이지를 새로 만들고, 헤더 `지원금/혜택` 링크도 목록 페이지로 연결했습니다.
- 블로그 생성 스크립트가 새 `events/items`, `benefits/items` 구조를 읽을 수 있도록 정리했습니다.

## 운영 정책 반영

- 혜택 데이터는 매일 오전 7시 30분(KST) 기준으로 조회하도록 워크플로우를 맞췄습니다.
- 중복 데이터는 `서비스ID` 기준으로 막도록 처리했습니다.
- 신청기한이 지난 혜택은 수집 단계에서 제외하고 저장본에서도 정리되도록 처리했습니다.
- 데이터가 동일하면 파일을 다시 쓰지 않도록 비교 로직을 추가했습니다.
