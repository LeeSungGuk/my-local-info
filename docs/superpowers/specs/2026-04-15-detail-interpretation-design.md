# Detail Interpretation Design

## Goal

이벤트 상세와 혜택 상세 페이지에 원문 데이터를 그대로 나열하는 대신, 현재 데이터 필드만으로 생성할 수 있는 해석형 안내 블록을 추가합니다.

## Scope

- 이벤트 상세 페이지에 `이런 분께 먼저 맞아요`, `방문 전에 확인할 것` 섹션 추가
- 혜택 상세 페이지에 `이 혜택을 먼저 볼 분`, `신청 전에 확인할 것` 섹션 추가
- 기존 데이터에 없는 사실은 만들지 않음
- 값이 없는 경우 안내 항목을 숨겨 `정보 없음` 문구를 더 늘리지 않음

## Approach

- 순수 함수 기반 규칙 유틸을 새로 만들어 이벤트/혜택 상세 데이터에서 해석 문구 배열을 생성합니다.
- 상세 페이지는 이 유틸이 반환한 결과만 렌더링합니다.
- 규칙은 현재 데이터가 주는 신호만 사용합니다.
  - 이벤트: 무료 여부, 대상, 일정, 장소, 운영 시간, 상세 링크 존재 여부
  - 혜택: 대상, 신청 방식, 상시 신청 여부, 접수기관, 문의처, 온라인/상세 링크 존재 여부

## Constraints

- 데이터가 비어 있는 항목은 과장해서 채우지 않음
- UI는 기존 상세 페이지 톤을 유지
- 변경 범위는 유틸 1개, 테스트 1개, 상세 페이지 2개로 제한

## Success Criteria

- 이벤트/혜택 상세에서 해석형 블록이 보인다.
- 문구가 기존 필드와 모순되지 않는다.
- 테스트로 대표 규칙 조합을 검증한다.
- `npm run build`가 통과한다.
