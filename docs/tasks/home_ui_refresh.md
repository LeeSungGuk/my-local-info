# 홈 UI Refresh 작업 계획

## 목표

홈 첫 화면에서 서울시티의 목적과 주요 탐색 경로를 더 빠르게 이해하게 만든다. 사용자가 검색을 여러 번 반복하지 않고도 상황별 코스, 행사, 혜택, 먹거리, 블로그 정보를 자연스럽게 탐색할 수 있게 한다.

## 배경

현재 PRD는 홈에서 서비스 목적 이해, 통합 검색, 상황별 코스, 추천 글, 오늘의 서울 요약, 행사·혜택 하이라이트를 제공하도록 정의한다. 이번 작업은 제품 범위나 기능 요구사항을 바꾸지 않고, 기존 요구사항을 더 명확한 UI/UX로 표현하는 개선 작업이다.

## 범위

- 홈 Hero 영역
- 홈 통합 검색 UI
- 상황별 코스 카드 섹션
- 홈 모바일 레이아웃과 터치 영역

## 제외 범위

- 검색 로직 변경
- 행사·혜택 목록 전체 리디자인
- 상세 페이지 리디자인
- 데이터 구조 변경
- 새 라이브러리 추가
- PRD 또는 SRS 요구사항 변경

## 작업 항목

1. Hero 문구와 CTA 흐름을 정리한다.
2. 통합 검색창의 시각적 위계와 검색 예시를 개선한다.
3. 상황별 카드가 주요 선택지처럼 보이도록 카드 구조와 CTA를 정리한다.
4. 모바일에서 Hero, 검색창, 상황별 카드의 간격과 터치 영역을 확인한다.
5. 기존 링크 이동과 검색 이동이 유지되는지 확인한다.

## 디자인 원칙

- 모바일 우선으로 읽기 쉽고 누르기 쉽게 만든다.
- 장식보다 제목, 요약, CTA의 정보 위계를 우선한다.
- 공공데이터 기반 서비스의 신뢰감을 유지한다.
- 한국어 문구는 기존 서울시티 톤을 유지하되 더 자연스럽고 짧게 다듬는다.
- 전체 리디자인이 아니라 홈 첫인상 개선에 집중한다.

## 완료 기준

- 홈 첫 화면에서 서울시티의 목적이 더 명확하게 보인다.
- 통합 검색창이 핵심 액션으로 인식된다.
- 상황별 카드가 주요 탐색 경로로 보인다.
- 모바일에서 주요 CTA의 터치 영역이 충분하다.
- 기존 검색 이동과 상황별 링크 이동이 깨지지 않는다.
- `npm run lint`와 `npm run build`가 통과한다.

## 검증 계획

- `npm run lint`
- `npm run build`
- 브라우저에서 `/` 확인
- 모바일 폭에서 Hero 줄바꿈, 검색창 높이, 카드 CTA 확인
- 검색어 입력 후 `/search?q=...` 이동 확인
- 상황별 카드의 통합 검색 링크와 기준 읽기 링크 확인
